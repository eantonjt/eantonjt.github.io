Git commands overview
This post simply details some git commands that I have learned about when reading the "Git pro book". This is mainly a summary for myself and to get a reason to explore these commands more, but hopefully others can find the notes useful as well.

git clone GITREMOTE NEWNAMEName the folder when you clone it
git clone https://github.com/libgit2/libgit2 myNewFolderName #Example
git push origin my_local_branch:my_desired_remote_branchPush local changes on a branch to a different branch on remote
git fetch -allFetches from all branches from the remotes
When I first started out with git I was not aware that origin/branch was different from branch. But if we just see it as two different branches then it is easy to understand why we should not always do git pull, and instead do git fetch and then sole any merge conflicts that appear etc.
Note that git clone does not need a remote does not need to be on another computer, you can clone from a shared drive or your own harddrive as git clone path/to/project.git and we can add it as a remote with git remote add local_proj path/to/project.git
Some useful flags for git commit
GIT CLI
üìù Copy
git commit -a -m "Adding changes made to all untracked files" Skipping git add . for all TRACKED files with the -a flag
git commit --amend Allows you to change the commit message of the last local commit that you did. Do not force-push amended commits since it ruins history for collaborators! You can also add new files to the commit if you add these to your staging area and then write git commit --amend
It can be good to ensure there are no whitespace errors before you commit, you can do this with git diff --check
Commands to make git log output in different format
GIT CLI
üìù Copy
git log -2 #Only shows last 2 commits
git log -p Shows the diff introduced by each commit
git log ‚ÄìS SearchText Lists the commits which added/removed the string "SearchText" to any file
git log ‚Äì-grep SearchText Searches through which commit messages contain a string "SearchText"
git log ‚Äì-since=2.hours Shows git commits made in the last 2 hours
git log --since="2 days 1 minute" Shows git commits made in the last 2 days and 1 minute
git log --until=2.hours Shows git commits from start until 2 hours ago
git log FILEPATH Shows only git commits that modified file at FILEPATH
git --author=AUTHOR Shows only git commits made by AUTHOR
git log --oneline Concise log without author info etc
Note that we can combine multiple of these flags together, to for example search for strings in commit messages and then show the diffs introduced by those commits etc. By default, git log will only show commit history for the checkout branch, use the flag --all to see the log for all branches.

Some other notes
Below are some definitions for some terminology that appears in Git.
  • Author: The person who originally wrote the work
  • Committer: The person that commited the work (This person can be different than the author if the author, e.g., sends the changes as a patch to the committer)

In VSCode one can easily handle merge conflicts. If one wants to merge a branch into another branch then just follow these steps:
  • Go to the branch CURRENT_BRANCH that is supposed to get the new changes
  • Press Ctrl-Shift-p and choose merge, then merge from THE_OTHER_BRANCH
  • If there is a conflict one will get a choice to accept incoming changes or what is currently there
  • When all conflicts have been resolved, add the file to the staging area and then commit.
Note you can do the same with remote branches but in some cases the Commit button can go away but then you can just type git commit in the terminal to finalize the merge.

VSCode also has limited functionality for partial staging. Go to the staging area and then select the regions that you want to commit and select "Stage selected ranges" in the top right.

Git rebasing should not be done on commits that others have based work on. This is since rebasing will change the corresponding commit hashes and make it difficult for others to merge later. Because for them, a certain history will have happened but it will be different from what the remote says happened.
